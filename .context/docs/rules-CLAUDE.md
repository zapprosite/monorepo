# Claude Monorepo Development Rules (`rules-CLAUDE.md`)

This document defines the development standards, architectural constraints, and coding patterns for the monorepo. Following these rules ensures consistency across the React frontend, Fastify backend, and shared TypeScript packages.

## 📋 General Principles

- **Schema-First**: All data structures must originate from `packages/zod-schemas`.
- **Absolute Type Safety**: Avoid `any` or `unknown`. Use Zod for runtime validation and TypeScript for static analysis.
- **Deep Imports**: When importing from shared packages (like `@repo/zod-schemas`), import the specific file to optimize tree-shaking (e.g., `import { ... } from '@repo/zod-schemas/user.zod'`).
- **Standardized Naming**:
    - Databases: `snake_case` for tables and columns.
    - Code: `camelCase` for variables/functions, `PascalCase` for Classes/Interfaces.

---

## 🏗 Backend Architecture (`apps/api`)

The backend is built with **Fastify**, **tRPC**, and **Orchid ORM**.

### 1. Database & ORM
- **Table Location**: Define tables in `apps/api/src/modules/<feature>/tables/`.
- **Registration**: Register all new tables in `apps/api/src/db/db.ts`.
- **Primary Keys**: Avoid generic `id` where ambiguity is possible; prefer explicit IDs in complex modules.
- **BaseTable**: Always extend the `BaseTable` class to inherit standard metadata and project-wide configurations.

### 2. tRPC Procedures
- **Public**: Use `publicProcedure` for auth-less endpoints (login, status).
- **Protected**: Use `protectedProcedure` for endpoints requiring a valid session. These automatically verify the session against the `SessionTable`.
- **Context**: Use the `ctx` object to access the current `user`, `teamId`, and `session` metadata.

### 3. Error Handling
Always use the `AppError` class located in `apps/api/src/middlewares/errorHandler.ts`:
```typescript
throw new AppError('Resource not found', 404);
```

## 💻 Frontend Architecture (`apps/web`)

The frontend is a **React 19** application powered by **Vite** and **MUI**.

### 1. Data Fetching
Use the `trpc` hooks for all API interactions. Never use `fetch` or `axios` directly for internal API calls.
```typescript
const { data, isLoading } = trpc.clients.getById.useQuery({ clientId });
```

### 2. Form Management
Use **React Hook Form (RHF)** along with the shared components in `packages/ui/src/rhf-form`.
- Use `useRhfForm` for standardized form initialization.
- Wrap forms in `RhfFormProvider`.

### 3. Icons
Use the centralized icon library in `packages/ui/src/icons` to ensure visual consistency.

## 📦 Shared Packages

### `packages/zod-schemas`
This is the **Source of Truth**.
1. Define the core schema.
2. Export specific types for different lifecycle stages:
   - `SelectAll`: For reading data.
   - `CreateInput`: For insertions (omits autogenerated fields like `createdAt`).
   - `UpdateInput`: For partial updates.

### `packages/ui`
- Components should be **presentational** ("dumb").
- Business logic is strictly forbidden in this package.
- Use MUI as the base and extend using the `theme` configuration.

## 🛠 AI and RAG Utilities (`scripts/hvac-rag`)

The repository contains specialized Python tools for HVAC-specific RAG (Retrieval-Augmented Generation).
- **Naming**: Python functions use `snake_case`.
- **Logic**: Tools like `hvac_rag_pipe.py` handle context building and LLM prompt construction for technical hvac documentation.

## 🚫 Development Constraints

1. **No Barrel Exports**: Never use `export * from './dir'` in `index.ts`. This breaks tree-shaking for the frontend.
2. **Constraint Validation**: Do not rely on database constraints alone; validate logic in the tRPC layer using Zod.
3. **Session Hijacking Prevention**: All protected routes must validate `UserAgent` and `IP Address` via the `sessionSecurity.middleware.ts`.
4. **Consistency**: Use `yarn format` (Biome) before every commit. Non-formatted code will fail CI.

## 🚀 Common Workflows

### Adding a New Database Table
1. Define schema in `packages/zod-schemas`.
2. Create table class in `apps/api/src/modules/<feature>/tables/`.
3. Register table in `apps/api/src/db/db.ts`.
4. Run `yarn run db g <name>` to generate migration.
5. Run `yarn run db up` to apply.

### Creating a New API Endpoint
1. Add the procedure to a new or existing router in `apps/api/src/modules/<feature>/`.
2. Connect the router to the `AppTrpcRouter` in `apps/api/src/routers/trpc.router.ts`.
3. Consume the new endpoint in `apps/web` using `trpc.<feature>.<method>.useQuery()`.
