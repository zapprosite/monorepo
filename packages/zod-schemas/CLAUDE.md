# Zod Schemas

Shared validation schemas for backend/frontend consistency.

## Entity Pattern

Each entity: `<entity>.zod.ts`
- `<entity>MandatoryZod` — Required fields
- `<entity>OptionalZod` — Optional/nullable
- `<entity>AutoGeneratedZod` — DB-generated (IDs, timestamps)
- `<entity>CreateInputZod` — Create operation
- `<entity>UpdateInputZod` — Update operation
- `<entity>SelectAllZod` — Full entity
- Query schemas: `<entity>GetByIdZod`, etc.

## Naming
- IDs: `userId`, `postId` (not generic `id`)
- FKs: `authorUserId` (descriptive)
- Schemas: camelCase + Zod suffix
- Types: PascalCase

## Imports

**External (other apps):**
```typescript
import { userCreateInputZod } from '@connected-repo/zod-schemas/user.zod'
```

**Internal (this package):**
```typescript
import { zString } from './zod_utils.js'  // Relative path
```

## Validators

| Validator | Use |
|-----------|-----|
| zString | Base string (trimmed) |
| zVarchar(min, max) | Variable-length string |
| zText(min) | Long text |
| zPrice | Decimal(10,2), min 0.01 |
| zQuantity | Decimal(11,3) |
| zAmount(min, max) | Financial amounts |
| zGSTIN | India GSTIN (15 chars) |
| zPAN | India PAN (10 chars) |
| zPhoneNumber | International format |
| zTimestamps | { createdAt, updatedAt } |

## Usage
```typescript
const schema = z.object({
  name: zVarchar(3, 100),
  price: zPrice,
})
export type Product = z.infer<typeof schema>
```

## Adding Entity
1. Create `src/<entity>.zod.ts`
2. Add table in apps/api/src/db/tables/
3. Register in db/db.ts
4. Import in router.trpc.ts
5. Run `yarn build`
