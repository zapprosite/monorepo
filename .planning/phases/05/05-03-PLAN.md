# Plan 05-03 - Grounding, citation and embedding smoke validation

<objective>
Convert the current dual-index implementation into a verifiable proof-first retrieval path by testing embedding health, FAQ provenance, raw evidence recovery, citation metadata and no-evidence fallback behavior.
</objective>

## Objective

Resolve the verification gap recorded in `05-02-SUMMARY.md`: the real RAG smoke ran but embeddings at `:4018` returned HTTP 400, so no raw evidence/citation could be validated. This plan adds health diagnostics and grounding tests before Phase 5 can ship.

1. Add a diagnostic helper for embedding calls that records:
   - backend URL,
   - model,
   - HTTP status,
   - sanitized error class,
   - embedding dimension when successful.
2. Add CLI flag `--embedding-health` to run a one-shot embedding smoke without starting the API server.
3. On HTTP 400 from `:4018`, print a safe actionable error that does not expose secrets.
4. Preserve current fallback behavior for normal chat requests.

<acceptance_criteria>

- Error output includes backend URL/model/status but no API key.
- Unit tests cover success, HTTP 400 and timeout paths.
</acceptance_criteria>

## Success Criteria

- The embedding HTTP 400 failure is diagnosable with a safe CLI command.
- Dual search is proven with mocked FAQ/raw Qdrant responses.
- Real smoke is allowed to fail only as an explicit environment/backend blocker, not as an ambiguous retrieval failure.
- Phase 5 cannot be marked ship-ready unless citations/page or chunk metadata are present in retrieved raw evidence.
