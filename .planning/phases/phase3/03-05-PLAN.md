# Plan 03-05 - Coverage feedback loop and report-only automation

<objective>
Close the loop from coverage gaps to acquisition attempts and back to an updated coverage report without requiring indexing side effects in report-only mode.
</objective>

## Objective

Add an automation mode that reads coverage/missing-manual data, attempts acquisition in a bounded/report-only way, records outcomes, and regenerates coverage/pending-review artifacts for operators.

1. Add `--report-only` mode to the expansion/acquisition pipeline.
2. In report-only mode:
   - read missing models,
   - build/search/rank candidates,
   - optionally validate download with `--dry-run`,
   - never call intake `--index`,
   - write a summary report.
3. Add `--limit` and `--brand` controls for safe batch execution.
4. Preserve checkpoint behavior for resumable batches.

<acceptance_criteria>
- Report-only mode cannot index documents.
- Limit/brand filters bound the number of acquisition attempts.
- Checkpoint records attempted model IDs and candidate outcomes.
</acceptance_criteria>
</task>
