# Plan 03-03 - PDF downloader validation and duplicate guard

<objective>
Turn ranked manual candidate URLs into validated PDF files with content-type, size, hash and duplicate protection before intake.
</objective>

## Objective

Implement the download/validation layer between `hvac_manual_finder.py` candidates and `hvac_add_manual.py`. The downloader must reject unsafe/non-PDF responses, avoid duplicate files by SHA-256, and produce machine-readable results for the acquisition pipeline.

## Tasks

<task type="auto" tdd="true">
<name>Task 1: Implement PDF download validation core</name>
<read_first>
scripts/hvac-rag/hvac_manual_downloader.py
scripts/hvac-rag/hvac_manual_finder.py
.planning/phases/phase3/03-CONTEXT.md
</read_first>
<files>scripts/hvac-rag/hvac_manual_downloader.py, tests/hvac-rag/test_manual_downloader.py</files>

1. Add a `DownloadResult` data model with:
   - `status`
   - `url`
   - `output_path`
   - `sha256`
   - `size_bytes`
   - `content_type`
   - `reason`
2. Implement `validate_pdf_response(headers, content)`:
   - Accept `application/pdf` and octet-stream responses whose bytes start with `%PDF`.
   - Reject HTML, empty content, non-PDF bytes and files below a configurable minimum size.
3. Implement deterministic output paths under `/srv/data/hvac-rag/incoming/pdf/{brand}/`.
4. Add CLI flags:
   - `--url`
   - `--brand`
   - `--model`
   - `--min-bytes`
   - `--output-json`
   - `--dry-run`
5. Never log tokens, query strings containing credentials, or response bodies.

<acceptance_criteria>
- `pytest tests/hvac-rag/test_manual_downloader.py -q` passes.
- HTML content is rejected with a traceable reason.
- Valid PDF bytes produce a SHA-256 and target path.
- `--dry-run` prints the planned target without network writes.
</acceptance_criteria>
</task>

<task type="auto" tdd="true">
<name>Task 2: Add duplicate guard by SHA-256</name>
<read_first>
scripts/hvac-rag/hvac_manual_downloader.py
scripts/hvac-rag/hvac_add_manual.py
</read_first>
<files>scripts/hvac-rag/hvac_manual_downloader.py, tests/hvac-rag/test_manual_downloader.py</files>

1. Before writing a downloaded PDF, compute SHA-256 from the response bytes.
2. Check existing PDFs under the incoming directory for the same SHA-256.
3. If a duplicate exists, return `status="duplicate"` with `output_path` pointing to the existing file.
4. Preserve idempotency: repeated downloads of the same content must not create multiple files.

<acceptance_criteria>
- Duplicate content is detected without rewriting a second file.
- Unique content is written exactly once.
- Result JSON includes `sha256`, `status`, `reason` and `output_path`.
</acceptance_criteria>
</task>
