# Plan 04-03 - endpoint/CLI de intake com validação de imagem e hints

<objective>
Expor a funcionalidade de visão HVAC via API REST e CLI, permitindo que fotos de campo sejam processadas, classificadas e integradas ao estado da conversa de forma automatizada.
</objective>

## Tasks

<task type="auto">
<name>Task 1: Endpoint de Visão no Pipe Python</name>
<files>scripts/hvac-rag/hvac_rag_pipe.py</files>

1. Importar `hvac_vision` em `hvac_rag_pipe.py`.
2. Criar endpoint `POST /v1/vision/intake`.
3. O endpoint deve receber `image` (base64), `hints` (opcional) e `session_id`.
4. Processar a imagem usando `hvac_vision.extract_from_image`.
5. Se bem-sucedido, atualizar o estado da sessão usando `hvac_memory_context.save_session_state`.
</task>

<task type="auto">
<name>Task 2: Exposição na API Node.js</name>
<files>apps/api/src/routes/hvac.schema.ts, apps/api/src/routes/hvac.client.ts, apps/api/src/routes/hvac.routes.ts</files>

1. Definir schemas de request/response para vision em `hvac.schema.ts`.
2. Implementar `callHvacVision` em `hvac.client.ts`.
3. Adicionar rota `POST /api/hvac/vision` em `hvac.routes.ts`.
</task>
