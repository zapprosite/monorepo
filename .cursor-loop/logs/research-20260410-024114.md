# pnpm Version Mismatch — Resumo

## 1. O que causa o erro

**Version mismatch** no pnpm ocorre quando:

| Tipo | Causa |
|------|-------|
| **Lockfile divergente** | `pnpm-lock.yaml` foi gerado por outra versão do pnpm (ex: v6 vs v8) |
| **Engine mismatch** | `package.json` especifica `engines` (node/pnpm) incompatível com a instalação atual |
| **Peer dependency conflict** | Dependências requerem versões conflitantes da mesma lib |
| **Workspace version mismatch** | `workspace:*` no `package.json` não encontra a versão no workspace |

---

## 2. Soluções comuns

### Atualizar o lockfile
```bash
pnpm install --lockfile-only
```
Isso atualiza o `pnpm-lock.yaml` sem baixar dependências.

### Ignorar engine strict
```bash
pnpm install --ignore-engines
```
Ou no `.npmrc`:
```ini
engine-strict=false
```

### Forçar atualização de lockfile incompatível
```bash
rm pnpm-lock.yaml
pnpm install
```

### Configurar peer dependency rules
No `pnpm-workspace.yaml` ou `.npmrc`:
```yaml
peerDependencyRules:
  allowedVersions:
    react: "17"
  allowAny:
    - "@babel/*"
```

## 3. Exemplo de erro comum

**ERR_PNPM_OUTDATED_LOCKFILE:**
```
ERR_PNPM_OUTDATED_LOCKFILE: Lockfile is up to date, but pnpm lockfile version is too old
```

**Solução:**
```bash
pnpm install
# ou
rm pnpm-lock.yaml && pnpm install
```

**ERR_PNPM_UNSUPPORTED_ENGINE:**
```
ERR_PNPM_UNSUPPORTED_ENGINE: Unsupported engine
```

**Solução:**
```bash
pnpm install --ignore-engines
# ou atualizar node/pnpm
```

---

## 4. Prevenção

1. **Commitar** `pnpm-lock.yaml` junto com `package.json`
2. **Usar `packageManager` no `package.json`** (Corepack):
   ```json
   {
     "packageManager": "pnpm@8.15.0"
   }
   ```
3. **Configurar `engineStrict: false`** em `pnpm-workspace.yaml` se não precisar de verificação rígida de engine
