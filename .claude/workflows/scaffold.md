# /scaffold — Gerador de Módulo Full-Stack

> Recebe um nome de entidade (ex: `products`, `invoices`, `tasks`) e gera todos os arquivos necessários.

---

## Passo 0 — Coletar informações

Perguntar ao usuário:
1. **Nome da entidade** (snake_case plural, ex: `blog_posts`, `invoices`)
2. **Campos principais** (nome, tipo, obrigatório/opcional)
3. **ID type**: `z.ulid()` (padrão) ou `z.uuid()`

Derivar automaticamente:
- `entityName` = snake_case singular (`blog_post`)
- `EntityName` = PascalCase (`BlogPost`)
- `entityNames` = snake_case plural (`blog_posts`)
- `entityId` = `${camelCase}Id` (`blogPostId`)

---

## Passo 1 — Zod Schema

**Arquivo:** `packages/zod-schemas/src/[entityName].zod.ts`

Seguir o padrão exato de `journal_entry.zod.ts`:

```typescript
import z from "zod";
import { zString, zTimestamps } from "./zod_utils.js";

export const [entityName]MandatoryZod = z.object({
  // campos obrigatórios informados pelo usuário
});

export const [entityName]OptionalZod = z.object({
  // campos opcionais informados pelo usuário (com .nullable())
});

const [entityName]AutoGeneratedZod = z.object({
  [entityId]: z.ulid(), // ou z.uuid()
});

export const [entityName]CreateInputZod = [entityName]MandatoryZod
  .extend([entityName]OptionalZod.partial().shape);
export type [EntityName]CreateInput = z.infer<typeof [entityName]CreateInputZod>;

export const [entityName]UpdateInputZod = [entityName]MandatoryZod
  .extend([entityName]OptionalZod.shape)
  .extend(zTimestamps)
  .partial();
export type [EntityName]UpdateInput = z.infer<typeof [entityName]UpdateInputZod>;

export const [entityName]SelectAllZod = [entityName]MandatoryZod
  .extend([entityName]OptionalZod.shape)
  .extend(zTimestamps)
  .extend([entityName]AutoGeneratedZod.shape);
export type [EntityName]SelectAll = z.infer<typeof [entityName]SelectAllZod>;

export const [entityName]GetByIdZod = [entityName]AutoGeneratedZod;
export type Get[EntityName]ByIdInput = z.infer<typeof [entityName]GetByIdZod>;
```

## Passo 2 — Orchid ORM Table

**Arquivo:** `apps/backend/src/modules/[entityNames]/tables/[entityName].table.ts`

Seguir o padrão das tables existentes em `apps/backend/src/db/tables/`. Incluir:
- `timestampNumber` para `createdAt`/`updatedAt`
- ID como `ulid()` ou `uuid()` com `.primaryKey()`
- FK relations se necessário
- Indexes para campos de query frequente

**Registrar a table:**
Após gerar o arquivo, instruir o usuário a adicionar em `apps/backend/src/db/db.ts`:
```typescript
import { [EntityName]Table } from "@backend/modules/[entityNames]/tables/[entityName].table";
// Adicionar ao objeto db:
[entityNames]: () => new [EntityName]Table(),
```

## Passo 3 — tRPC Router

**Arquivo:** `apps/backend/src/modules/[entityNames]/[entityNames].trpc.ts`

Seguir o padrão de `journal_entries.trpc.ts`. Gerar procedures:
- `getAll` — query, `protectedProcedure`
- `getById` — query com `[entityName]GetByIdZod`
- `create` — mutation com `[entityName]CreateInputZod`
- `update` — mutation com `[entityName]UpdateInputZod`
- `delete` — mutation com `[entityName]GetByIdZod`

**Registrar o router:**
Instruir adicionar em `apps/backend/src/routers/trpc.router.ts`:
```typescript
import { [entityNames]RouterTrpc } from "@backend/modules/[entityNames]/[entityNames].trpc";
// Dentro do appRouter:
[entityNames]: [entityNames]RouterTrpc,
```

---

## Passo 4 — Migration

Instruir o usuário a rodar:
```bash
yarn db g create_[entityNames]_table
yarn db up
```

## Passo 5 — Frontend Module

**Arquivo:** `apps/frontend/src/modules/[entityNames]/[entityNames].router.tsx`

Seguir o padrão de `journal-entries.router.tsx`. Gerar rotas React Router:
```typescript
export const [entityNames]Routes = [
  { path: "/[entityNames]", element: <[EntityName]ListPage /> },
  { path: "/[entityNames]/:id", element: <[EntityName]DetailPage /> },
];
```

**Arquivo:** `apps/frontend/src/modules/[entityNames]/pages/[EntityName]List.page.tsx`

Componente React com:
- `trpc.[entityNames].getAll.useQuery()`
- Listagem básica com loading/error state
- Link para detalhe
