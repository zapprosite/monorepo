# Memory Operations

## Como estruturar contexto persistente

### Formato recomendado de registro
```
[CATEGORIA] [Conteúdo]
Exemplos:
[WORK] CTO de startup de pagamentos no Brasil
[STACK] Node.js, TypeScript, PostgreSQL, Docker
[PREF] Prefere respostas diretas sem rodeios
[PROJECT] Construindo feature de checkout em Jan/2025
[GOAL] Lançar MVP até fim do trimestre
```

### Categorias sugeridas
- WORK: cargo, empresa, área de atuação
- STACK: tecnologias preferidas e em uso
- PREF: preferências de comunicação e formato de resposta
- PROJECT: projeto atual e contexto relevante
- GOAL: objetivo de curto/médio prazo
- CONTEXT: qualquer outro contexto relevante

## Como recuperar e aplicar contexto

### Ao iniciar uma conversa
1. Verifique se há contexto salvo relevante para a tarefa
2. Aplique silenciosamente sem mencionar que está usando memória
3. Se o contexto estiver desatualizado, pergunte brevemente

### Ao detectar contexto novo na conversa
1. Identifique se é informação nova ou atualização de algo existente
2. Registre mentalmente para aplicar no restante da conversa
3. Se for informação importante e duradoura, sugira salvar

## Privacidade e limites
- Não tente inferir informações que não foram ditas explicitamente
- Nunca registre informações financeiras específicas, senhas ou dados pessoais sensíveis
- Se o usuário pedir para esquecer algo, confirme e remova do contexto
