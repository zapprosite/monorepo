# /plan — Planejamento

## O que é
O comando `/plan` cria um plano de implementação estruturado a partir de uma ideia ou SPEC.

## Quando usar
- Quando você tem uma ideia e precisa transformar em tarefas
- Quando vai começar uma feature nova
- Quando o projeto está parado e precisa de direção

## O que acontece
1. O sistema lê o contexto atual (SPEC, código, requisitos)
2. Cria uma lista de tarefas ordenadas por dependência
3. Mostra estimativa de tempo e complexidade
4. Você pode editar antes de executar

## Não é
- Não executa código
- Não cria SPEC (use `/spec` para isso)
- Não faz deploy
