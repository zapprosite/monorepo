# /universal-debug — Debug Sistemático

## O que é
O comando `/universal-debug` segue 4 fases sistemáticas: Reproduce → Isolate → Understand → Fix.

## Quando usar
- Quando você tem um bug e não sabe a causa
- Quando algo parou de funcionar
- Quando precisa investigar problema

## As 4 fases

### 1. Reproduce (Reproduzir)
- Encontrar comando/maneira de触发 o bug
- Confirmar que o problema existe
- Documentar passos exatos

### 2. Isolate (Isolar)
- Identificar qual componente causa o problema
- Eliminar código que não é relacionado
- Encontrar o ponto exato da falha

### 3. Understand (Entender)
- Analisar por que falha
- Verificar histórico de mudanças
- Entender o fluxo completo

### 4. Fix (Corrigir)
- Aplicar correção
- Verificar que bug não existe mais
- Testar casos relacionados
