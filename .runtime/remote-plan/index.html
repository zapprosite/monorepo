Plano proposto para acesso remoto seguro

Objetivo: acessar a tela do Ubuntu Desktop remotamente sem expor desktop, VNC, RDP, noVNC ou portas sensíveis em subdomínio público aberto.

Tailscale privado RustDesk Sem DNS novo Link público só para este plano

Arquitetura recomendada

Cliente remoto
  |
  v
Tailscale VPN
  |
  v
Host Ubuntu Desktop
  |
  v
RustDesk com Direct IP

Regra de segurança

  • Nao publicar espelhamento de tela direto no Cloudflare.
  • Nao abrir VNC, RDP, noVNC ou Chrome debugging na Internet.
  • Manter acesso real restrito a Tailscale.
  • Usar Cloudflare apenas para apps/documentacao protegida.

Implementação

  • Usar o projeto existente pc-cel/rustdesk-ubuntu-247.
  • Validar Tailscale autenticado e host online.
  • Garantir RustDesk instalado, rodando e com autostart.
  • Ativar Direct IP no RustDesk.
  • Liberar a porta do RustDesk apenas na interface Tailscale.

Critérios de sucesso

  • Cliente remoto vê o host na tailnet.
  • RustDesk aceita conexão via rede privada.
  • Senha permanente configurada fora desta página.
  • Nenhum subdomínio novo criado para tela.
  • Nenhuma porta de desktop aberta publicamente.

Comandos de validação no host

tailscale status
tailscale ip -4
rustdesk --get-id
pgrep -a rustdesk
ss -tulnp | grep 21118
sudo ufw status verbose
